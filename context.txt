EIGENCLASS: Automated Attendance System Using a Linear Algebra Pipeline
Problem Statement
Educational institutions face challenges with manual attendance marking—it is time-consuming, prone to errors, and vulnerable to proxy attendance. In this mini project, you will build an automated attendance system using face recognition with a structured linear algebra pipeline starting from raw face images and ending with student identification and attendance logging. The goal is to identify a student from their face image and mark their attendance automatically.

Linear Algebra Flow Applied to Face Recognition Attendance
Step 1: Real-World Data to Matrix Representation
We collect face images of students. Each grayscale image of size M×N pixels is flattened into a column vector of length d = M×N. Multiple training images from all students are arranged as rows in a matrix.

Topics Used:

Matrices

Vectors

Data Representation

Step 2: Computing the Average Face
We calculate the mean of all face vectors in the training set. This "mean face" represents the common features across all students. Each individual face is a variation away from this average.

Topics Used:

Mean Vector

Linear Combinations

Step 3: Centering the Data
We subtract the mean face from every face vector. This shifts the data so that the average becomes zero. The centered data reveals how each face differs from the common average.

Topics Used:

Vector Subtraction

Data Normalization

Step 4: Constructing the Covariance Matrix
We build the covariance matrix from the centered face vectors. This matrix captures how different pixel positions vary together across all faces—for example, if the left eye darkens, does the right eye also darken?

Topics Used:

Covariance Matrix

Matrix Multiplication

Step 5: Finding Eigenfaces Using Eigen decomposition
We compute the eigenvalues and eigenvectors of the covariance matrix. Each eigenvector, when reshaped back into an image, reveals a fundamental facial feature called an "Eigenface." The largest eigenvalues correspond to the most important variations among faces (such as lighting direction or face shape).

Topics Used:

Eigenvalues

Eigenvectors

Eigen decomposition

Step 6: Dimensionality Reduction Using Principal Component Analysis (PCA)
We select only the top k eigenvectors (those with the largest eigenvalues) to form the "face space." This reduces the problem from thousands of pixels to just 50–100 important dimensions, removing noise and improving computational efficiency.

Topics Used:

Principal Component Analysis (PCA)

Dimensionality Reduction

Step 7: Projecting Faces into Face Space
We project every training face onto the selected Eigenfaces to obtain a weight vector for each student. This weight vector is the compact representation of that student's face in the reduced face space.

Topics Used:

Projection onto a Subspace

Dot Product

Linear Transformation

Step 8: Recognizing a New Face Using Distance
For a new face image, we first center it (subtract the mean face) and project it onto the same Eigenface space to get its weight vector. We then compute the Euclidean distance between this new weight vector and all stored training weight vectors. The closest match identifies the student.

Topics Used:

Euclidean Distance

Nearest Neighbor

Distance Metrics

Step 9: Attendance Marking as a Linear Decision
We compare the minimum distance against a threshold. If the distance is below the threshold, the face is recognized and attendance is marked with a timestamp. If above, the face is marked as unknown. The final attendance sheet is generated as a record.

Topics Used:

Thresholding as a Decision Boundary

System of Linear Inequalities

Summary Table: Linear Algebra Pipeline
Step	Task	Linear Algebra Concept
1	Convert face images to numerical data	Matrices, Vectors
2	Find the average face	Mean Vector
3	Remove the average from all faces	Vector Subtraction
4	Capture how pixels vary together	Covariance Matrix
5	Extract fundamental face features	Eigenvalues & Eigenvectors
6	Reduce dimensions	PCA, Dimensionality Reduction
7	Get compact face representation	Projection onto Subspace
8	Match a new face to a student	Euclidean Distance
9	Mark attendance or reject	Decision Boundary
